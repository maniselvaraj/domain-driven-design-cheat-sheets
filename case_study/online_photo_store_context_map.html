DDD Standard Notation Reference
U (Upstream) / D (Downstream) Customer-Supplier relationship. Upstream defines the interface, downstream conforms. Arrow points downstream (direction of dependency).
ACL (Anticorruption Layer) Protects downstream context from upstream changes. Translates external models to domain concepts. Used for external integrations.
PL (Published Language) Well-documented shared language between contexts. Often implemented as domain events. Upstream publishes, multiple downstreams consume.
CF (Conformist) Downstream accepts upstream model as-is. No translation. Used when cost of ACL exceeds benefit (e.g., analytics consuming events).
OHS (Open Host Service) Upstream provides protocol for multiple downstreams. Combined with Published Language for event-driven architectures.
Bounded Context Explicit boundary where a specific model applies. Contains aggregates, entities, value objects with consistent ubiquitous language.
Aggregate Root Entry point for all operations on the aggregate. Enforces invariants. Controls transactional consistency boundary.
Value Object (VO) Immutable object defined by attributes. No identity. Shared concepts like Address, Money, DateRange.
Entity Object with identity that persists over time. Mutable. Identity matters more than attributes (e.g., OrderLine, CartItem).
Core Domain Strategic competitive advantage. Receives maximum investment. Contains complex business rules unique to organization.
Supporting Subdomain Necessary for operations but not competitive differentiator. Custom-built. Simpler than core domain.
Generic Subdomain Solved problem available off-the-shelf. Buy vs build decision. Examples: authentication, payment processing, notifications.
